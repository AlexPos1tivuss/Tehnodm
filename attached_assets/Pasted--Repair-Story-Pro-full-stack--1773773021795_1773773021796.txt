Проект: "Repair Story Pro" — диплом: полнофункциональное приложение для сервис-центра (full-stack).

Цель: Сгенерируй рабочий репозиторий, готовый к запуску на Replit. Проект — без каталога товаров и без ИИ. Показывает навыки full-stack: auth, база, REST API, realtime, календарь, экспорт PDF, тесты, Docker, README.

СТЕК (предпочтительный)
- Frontend: React + Vite + TypeScript + Tailwind CSS
- Backend: Node.js + Express + TypeScript
- ORM: Prisma + SQLite
- Realtime: Socket.io
- Auth: JWT + bcrypt
- Тесты: Jest + supertest, React Testing Library
- Dockerfile + .replit config

ТРЕБОВАНИЯ/ФУНКЦИИ
1) Auth:
   - Регистрация / логин (email + пароль). Роли: client, technician, admin.
   - Hashing паролей (bcrypt), JWT access token (15m), refresh token (opt).
2) Bookings:
   - Клиент создаёт заявку: device, issue, предпочитаемое время (опционально).
   - Генерация кода: R-xxxxx (последние 5 цифр timestamp + случай).
   - Сохранение в bookings (SQLite).
3) Calendar:
   - Простая логика слотов (например 30-мин слоты 9:00—18:00). При брони слот помечается занятым.
   - API /api/calendar/slots?date=YYYY-MM-DD возвращает доступные/занятые слоты.
4) Workflow & Logs:
   - Статусы: new -> accepted -> diagnosing -> repairing -> ready -> closed.
   - При смене статуса создаётся запись в repair_logs (from, to, note, by).
   - Админ/техник может назначить заявку (technicianId).
5) Realtime:
   - Socket.io канал: при смене статуса emit событие booking:update (bookingId, newStatus, log).
   - Клиент подписывается на событие по ID, видит обновления в реальном времени.
6) Public Tracker:
   - Публичная страница: ввод R-xxxxx → GET /api/bookings/:code → показывает текущий статус и timeline (repair_logs).
7) Troubleshooter:
   - Decision tree в data/tree.json (3–4 уровня). На frontend компонент, показывающий breadcrumb пути.
8) Admin Panel:
   - Таблица заявок с фильтром по статусу, назначение техника, смена статуса, экспорт CSV/PDF.
9) Export PDF:
   - Endpoint /api/export/:id/pdf (генерация PDF с данными заявки и логами); можно использовать pdfkit или html-pdf.
10) Tests:
   - Unit тесты для генератора ID, валидатора слотов.
   - Integration тесты API: создание booking, поиск по code, смена статуса.
11) Дополнительно:
   - Dockerfile, seed-скрипт (создаёт admin/technician/client), README с инструкциями запуска и деплоя в Replit.

ПРИМЕР API FLOW (пример запросов)
- POST /api/auth/login {email,password} -> {accessToken}
- POST /api/bookings {device,issue,appointmentAt} (Auth) -> {code: "R-12345", id}
- GET /api/bookings/R-12345 -> {booking, logs}
- PATCH /api/bookings/:id/status {to:'diagnosing', note:'начали диагностику'} (Auth tech/admin)
